# Gateway session memory (Mastra Memory + LibSQL)

The Phase 2 gateway REPL initially called `agent.generate()` with only the operator's latest line, so multi-turn chat within a session had no history — grill follow-ups, commit-message requests, and resume context were lost between turns. We added **Mastra Memory** (`@mastra/memory`) with **LibSQL** persistence (`@mastra/libsql`) so each gateway session passes a stable `{ thread, resource }` pair on every `generate()` call: `resource` is the operator (`operator`), `thread` is a new UUID per `npm run gateway` launch. Message history (last 40 turns) and **thread-scoped working memory** are stored in gitignored `.mastra/memory.db`. On session start, `gatewaySession.ts` creates the thread via `saveThread` then seeds working memory with role, tool catalog, and current work-item state; `resume #N` refreshes working memory when a work item is loaded. Cross-session continuity for engineering work remains on **disk** (PRDs in `docs/prds/`, work-item manifest in `.mastra/state/`) — memory is for within-session conversation, not a substitute for the "new thread per step" deliverable model in the Phase 2 north star. The trade-off is a new runtime dependency and local SQLite state; we accept this because without memory the gateway cannot sustain a grill or ship conversation, and LibSQL matches the local-first Mac mini target without introducing Postgres or cloud storage.

Considered alternatives: **pass manual message arrays in the gateway** without Mastra Memory — viable but reinvents persistence and working-memory patterns Memory already provides. **Resource-scoped working memory** (shared across all gateway threads for the operator) — rejected for Phase 2 to avoid bleeding context between unrelated sessions; thread scope keeps each `npm run gateway` launch isolated. **No memory; rely on disk artifacts only** — insufficient for multi-turn prompts within a single skill step (proven by the teal color and ship-doc failures before memory shipped). **Observational / semantic recall** — deferred; `lastMessages: 40` is sufficient for Phase 2 gateway sessions.
